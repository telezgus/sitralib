Tabla 4.6: Bits de Status I

VP: Verde del movimiento 1. 0 = Apagado, 1 = Encendido

PFL: Plan forzado local desde CC. 0 = Normal, 1 = Forzado

CP: Cambiando Plan. 0 = Normal, 1 = Cambiando

LP: Llave panel local central. 0 = Local, 1 = Central

TD: Tipo de Día. 0 = Normal, 1 = Feriado o evento

C: Centralizado. 0 = Local, 1 = Centralizado

AP: Apagado. 0 = Normal, 1 = LLave panel apagado o EC apagado

TIT: Titilante. 0 = Normal, 1 = LLave panel titilante o EC titilante

Bits
7 6 5 4 3 2 1 0
0 SI 0 0 0 0 0 AIS

Tabla 4.7: Bits de Status II

AIS: EC aislado de grupo (No acepta comandos grupales). 0 = Normal, 1 = Aislado

SI: EC en secuencia de inicio (intervalos A (34) y B (35)). 0 = Normal, 1 = Secuencia de inicio

Bits
7 6 5 4 3 2 1 0
0 0 0 0 0 0 D2 D1

Tabla 4.8: Bits de Status III

D1: Demanda 1. 0 = Desocupada, 1 = Ocupada

D2: Demanda 2. 0 = Desocupada, 1 = Ocupada

Bits
7 6 5 4 3 2 1 0
0 0 0 0 BT CV TSUP FR

Tabla 4.9: Bits de Alarmas

FR: Falta de rojo. 0 = Normal, 1 = Falla

TSUP: Tiempo suplementario de ciclo. 0 = Normal, 1 = Falla

CV: Conflicto de verde. 0 = Normal, 1 = Falla

BT: Baja Tensión. 0 = Normal, 1 = Falla

Si el grupo al que va dirigido el mensaje está mal, el EC no responde a pesar de que el Número de cruce es correcto.

Bit AIS (Aislado)

El bit Aislado indica que el EC no acepta comandos grupales. Este bit es siempre igual a cero, pero es uno cuando:

El EC se encuentra en falla

El EC recibe un comando Desagrupar

Cuando el EC recibe el comando Agrupar, seteará el bit Aislado a cero, siempre y cuando este no se encuentre en estado de error.

Bit SI (Secuencia de Inicio)

Este bit se setea siempre en cero, salvo que el EC se encuentre ejecutando la secuencia de inicio.
